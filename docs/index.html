Watch npm install.

`daytona-npm-watch` runs package installs in Daytona, captures logs, traces, and file snapshots, then emits a plain JSON report your team can read, demo, and extend.

03 / modes

Two ways to inspect the package.

observe

Run scripts in isolation

Installs with lifecycle scripts enabled and captures npm output, optional `strace`, and filesystem before/after snapshots.

npm install <pkg> --foreground-scripts --loglevel=silly
safe

Inspect without running scripts

Installs with scripts disabled, then scans every installed `package.json` for lifecycle hooks and suspicious command content.

npm install <pkg> --ignore-scripts --no-audit --no-fund

04 / detection

Explainable signals, not black-box scoring.

The prototype looks for behavior and strings that are useful in training demos. Each signal maps back to an artifact you can SSH into the sandbox and inspect.

postinstall node -e python -c sh / bash / zsh connect(...) curl / wget chmod base64 eval child_process .ssh .npmrc

05 / evidence

Keep the sandbox, inspect the scene.

With `--keep-sandbox`, your demo becomes interactive. SSH into Daytona and open the exact files named in the JSON report.

npm.logLifecycle output and npm debug noise
npm.straceProcess, file, and network syscalls when available
files.beforeFilesystem baseline before install
files.afterFilesystem state after install
project/The temporary npm project and node_modules tree

07 / boundaries

Built for defensive training.

The demo package is harmless. The detector is intentionally transparent and heuristic-based. It is not a complete malware sandbox, but it clearly shows how install-time behavior can be isolated, captured, and explained.
