02 / incident context

Why npm install behavior matters.

The Shai-Hulud npm supply-chain incident showed how package lifecycle scripts can turn dependency installation into an execution path. Once trusted package access was compromised, malicious releases could run during install and create a fast-moving blast radius.

What was vulnerable Trusted maintainer/package publishing paths, install-time scripts, and exposed automation tokens.
How it ignited Compromised package versions executed during `npm install`, searched for secrets, and attempted propagation.
Why detection helps Signals like `postinstall`, shell execution, network calls, and token-file access are visible in sandbox artifacts.

05 / modes

Two ways to inspect the package.

observe

Run scripts in isolation

Installs with lifecycle scripts enabled and captures npm output, optional `strace`, and filesystem before/after snapshots.

npm install <pkg> --foreground-scripts --loglevel=silly
safe

Inspect without running scripts

Installs with scripts disabled, then scans every installed `package.json` for lifecycle hooks and suspicious command content.

npm install <pkg> --ignore-scripts --no-audit --no-fund

06 / detection

Explainable signals, not black-box scoring.

The prototype looks for behavior and strings that are useful in training demos. Each signal maps back to an artifact you can SSH into the sandbox and inspect.

postinstall node -e python -c sh / bash / zsh connect(...) curl / wget chmod base64 eval child_process .ssh .npmrc
